# PRD: UX/UI polish ก่อน pitch

ขัดหน้าตาแอปให้คู่ควรกับระบบข้างใน · ทำ 20–22 ก.ค. แล้ว **freeze 23 ก.ค.** เพื่อให้ QA-02 จับเวลาบน UI ที่นิ่ง

ที่มา: เดินดูจอจริงทั้ง 7 หน้า × light/dark × มือถือ/เดสก์ท็อป (20 ก.ค.) — บันทึกสิ่งที่เจออยู่ในแต่ละ issue

## หลักการ

- **อบอุ่น ไม่ใช่เด็ก** — โจทย์ข้อ 8 สั่งว่าไม่กดดัน ไม่ตัดสิน แต่ธีมเดิมยืมจากเว็บการตลาด Cohere ซึ่งเย็นและเป็นทางการ · ทำให้ธีมตรงกับ**เสียงของแอปที่อบอุ่นอยู่แล้ว**
- **ห้ามใช้สีตัดสินพฤติกรรมผู้ใช้** — แดง/ส้มเตือนภัยสงวนไว้ให้ error ของระบบเท่านั้น
- **ไม่รื้อ layout** — ระบบ responsive เดิมใช้ได้จริง (sidebar, step rail, ตาราง/การ์ด) ปัญหาคือช่องว่างตาย ไม่ใช่โครงผิด
- **ไม่เพิ่ม npm package** — ของที่มีพอแล้ว

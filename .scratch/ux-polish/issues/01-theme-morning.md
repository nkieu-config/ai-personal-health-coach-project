# UX-01 — Theme "เช้าวันใหม่"

Status: ready-for-agent

เปลี่ยน design token ทั้งแอปให้อบอุ่นและตรงกับเสียงของแอป · แก้ `src/app/globals.css` เป็นหลัก

## ปัญหา

**1. ธีมทะเลาะกับคำพูดของแอป** — dashboard เขียนว่า *"วันนี้เดดไลน์ด้วย — บันทึกได้ในวันแบบนี้ก็ดีมากแล้ว ขอบคุณที่แวะมาดูแลตัวเอง"* แต่ห่อด้วยขาวล้วน `#ffffff` + เขียวเข้มเย็น `#003c33` ซึ่งยืมมาจากเว็บการตลาดของ Cohere — ออกแบบมาขายบริษัท ไม่ได้ออกแบบมาปลอบนักศึกษาที่กรอกตอนตี 1

**2. สีกราฟไม่สัมพันธ์กับสิ่งที่มันแทน** — ยกมาจากแบรนด์ Cohere ทั้งชุด ต้องอ่าน legend ทุกครั้ง

| token | เดิม | แทนอะไร | ปัญหา |
| --- | --- | --- | --- |
| `--chart-1` | `#003c33` เขียวเข้ม | นอน | ทำไมการนอนเป็นสีเขียว |
| `--chart-2` | `#1863dc` น้ำเงิน | กิน | ทำไมอาหารเป็นสีน้ำเงิน |
| `--chart-3` | `#ff5e3a` ส้มแดง | เคลื่อนไหว | **สีเตือนภัยใช้กับพฤติกรรมผู้ใช้ — ขัดโจทย์ข้อ 8** |

**3. `--ring: #4c6ee6`** ฟ้าที่ไม่เข้ากับอะไรในพาเลตต์เลย

## ทำอะไร

โหมดสว่าง — `background` ครีมอุ่นคือท่าที่ได้ผลมากที่สุด, การ์ดขาวลอยบนครีมได้ความลึกโดยไม่ต้องใช้เงาหนัก

| token | ใหม่ |
| --- | --- |
| `--background` | `#FDFAF5` ครีมอุ่น |
| `--card` / `--popover` | `#FFFFFF` |
| `--foreground` | `#2B2621` หมึกอุ่น |
| `--primary` | `#1D6B53` มัทฉะเข้ม |
| `--secondary` / `--muted` | `#F4EDE2` ทรายอุ่น |
| `--muted-foreground` | `#6B6156` |
| `--accent` / `--accent-foreground` | `#FFD9B8` / `#6B3F1D` แอปริคอต |
| `--border` | `#EBE3D6` |
| `--ring` | `#1D6B53` (เลิกใช้ฟ้าที่ไม่เข้าพวก) |
| `--destructive` | `#C2412C` — **ใช้กับ error ระบบเท่านั้น** |

สีกราฟให้ความหมายตรงกับสี → อ่านออกโดยไม่ต้องดู legend

| token | ใหม่ | เหตุผล |
| --- | --- | --- |
| `--chart-1` นอน | `#6B8FD4` ฟ้าคราม | กลางคืน สงบ |
| `--chart-2` กิน | `#E9A13B` เหลืองอำพัน | อาหาร ความอบอุ่น |
| `--chart-3` เคลื่อนไหว | `#46A883` เขียวสด | พลัง การเติบโต |
| `--chart-4` | `#A98CD1` ลาเวนเดอร์ | |
| `--chart-5` | `#E2836F` ปะการังนุ่ม | |

โหมดมืดอุ่นตาม — `--background: #1B1815` ถ่านอุ่น (เดิม `#17171c` อมน้ำเงิน) · `--primary: #5FBE99` มิ้นต์สว่างเด้งบนพื้นมืด

อัปเดต `themeColor` 2 บรรทัดใน `src/app/layout.tsx` ให้ตรงด้วย — เป็น hardcode สีจุดเดียวที่เหลือในโปรเจกต์

## ทำไมความเสี่ยงต่ำ

- ทั้งแอปมี hardcode สีแค่ 2 บรรทัด นอกนั้นใช้ token — **แก้ไฟล์เดียวกระจายทั้งแอป**
- `e2e` เช็ค contrast ≥ 4.5:1 ทุกหน้า × light/dark อยู่แล้ว → **ถ้าสีใหม่อ่านไม่ออก CI จับได้ก่อนกรรมการ**
- revert = `git revert` commit เดียว

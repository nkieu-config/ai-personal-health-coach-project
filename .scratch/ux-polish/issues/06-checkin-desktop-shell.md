## อาการ

คีนรู้สึกว่าหน้า /checkin บนเดสก์ท็อป "ไม่ค่อยเข้ากับส่วนอื่นของแอป" — วัดที่ 1440px
ด้วยเกณฑ์เดียวกันทั้ง 7 หน้าแล้วยืนยันว่าจริง:

| หน้า | h1 ที่มองเห็น | ไอคอน | ขอบซ้ายการ์ด |
| --- | --- | --- | --- |
| **/checkin** | **ไม่มี (`sr-only`)** | **ไม่มี** | **744** |
| /dashboard · /goals · /coach · /checkin/history | มี | มี | 336 |
| /reflection · /settings/privacy | มี | มี | 464 |

## 1. หน้าเดียวในแอปที่ไม่มีหัวข้อหน้า

`checkin/page.tsx:48` เป็น `<h1 className="sr-only">` — ทุกหน้าที่เหลือใช้สูตรเดียวกัน
คือ **ไอคอนสี primary + h1 + คำบรรยาย 1 บรรทัด** ซึ่งเป็นลายเซ็นที่ทำให้ผู้ใช้สแกนออกว่าอยู่หน้าไหน

ชื่อหน้าไปซ่อนอยู่ใน CardTitle (`เช็คอิน · กิน`) แทน ซึ่งเป็นระดับ component ไม่ใช่ระดับหน้า

`checkin/edit/[date]/page.tsx:20` มีอาการเดียวกัน

## 2. เนื้อหาไม่อยู่ในแนวกริดเดียวกับแอป

`checkin-form.tsx:306` ใช้ `lg:justify-center` → จับ rail + การ์ดมาวางกลางพื้นที่ 1184px
ขอบซ้ายเลยตกที่ 744 แทนที่จะเป็น 336 เหมือนหน้าอื่นที่ใช้ `PageContainer width="content"`
(max-w-5xl กลางจอ = ซ้ายที่ 336 พอดี)

ผลคือสลับหน้าไป-มาแล้วทั้งบล็อกกระโดดไปทางขวาอย่างเห็นได้ชัด

## สาเหตุราก

step-rail มาจาก INFRA-17 ที่แก้โจทย์ "ฟอร์มโล่งบนเดสก์ท็อป" — **แก้ถูกทาง**
(เติม panel ข้าง ๆ ไม่ขยายฟอร์ม ตาม `DESIGN.md`) แต่แก้ที่ระดับ component เท่านั้น
ไม่ได้แก้ที่ระดับหน้า จึงได้ rail มาโดยที่หน้ายังไม่มีหัวข้อ และทั้งบล็อกยังลอยกลางจอ

## ขอบเขต

- เพิ่มหัวข้อหน้าให้ /checkin และ /checkin/edit/[date] ตามสูตรเดียวกับหน้าอื่น
- ถอด `lg:justify-center` ให้ rail เริ่มที่แนว 336 เท่ากับ h1
- **คง `max-w-md` ของฟอร์มไว้** — `DESIGN.md` สั่งไว้ว่า "อย่าขยายฟอร์ม (แคบ = กรอกง่ายกว่า)"
  ปัญหาไม่ใช่ความกว้างการ์ด แต่คือตำแหน่งของทั้งบล็อก
- **ไม่แตะมือถือ** — แก้เฉพาะ `lg:` และการเพิ่มหัวข้อ (ซึ่งมือถือควรมีอยู่แล้ว)

## ไม่ทำ

**ย้าย "ขั้นที่ 1 จาก 4" ขึ้นไปอยู่กับหัวข้อหน้า** — เคยเสนอไว้ตอนวิเคราะห์ แต่ทำไม่ได้สะอาด
เพราะ `page.tsx` เป็น Server Component ส่วน `step` เป็น client state ของฟอร์ม
ถ้าจะย้ายต้องดัน state ขึ้นไปหรือทำ context ทั้งชุด — แลกไม่คุ้มกับกำไรที่ได้

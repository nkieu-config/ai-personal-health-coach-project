Refs: FR-5.1, F5-01, docs/00 §11 (Evaluation Criteria)

## ปัญหา

เกณฑ์ Personalization ในโจทย์เขียนว่า *"เชื่อมโยงกับ**ตารางชีวิต**และ**ข้อจำกัด**ของผู้ใช้"* และงานข้อแรกของ F5-01 ก็สั่งว่า *"ปรับตาม check-in + disruptor + **ข้อจำกัดจริงของผู้ใช้**"* — ส่วนข้อจำกัดยังไม่ได้ทำ

`buildGoalPrompt(situations, checkins)` ส่งให้ Gemini แค่สถานการณ์ + ตัวอย่าง check-in · **ไม่มีอะไรจากโปรไฟล์เลย** ทั้งที่ onboarding เก็บมาครบแล้วและ `getProfile()` ก็พร้อมใช้:

- `typical_constraints` — ไม่ค่อยมีเวลา / ไม่มีสถานที่ออกกำลังกาย / งบจำกัด / พักผ่อนไม่พอ / เดินทางนาน
- `busy_periods` — ช่วงสอบ / ส่งโปรเจกต์ / สิ้นเดือน / ทุกสัปดาห์ / ไม่แน่นอน
- `early_days` — วันที่ต้องตื่นเช้า

ผลคือความไม่สมมาตรที่อธิบายยากบนเวที: **โค้ชแชทรู้ข้อจำกัด** (`chat/context.ts` ส่งให้ครบ) แต่ **ระบบแนะนำเป้าหมายไม่รู้** — ถ้ากรรมการถามว่า "ผู้ใช้บอกว่าไม่มีสถานที่ออกกำลังกาย goal จะต่างไหม" ตอนนี้คำตอบคือไม่ต่าง

หลักฐานประกอบจาก QA-01 รอบ 16 ก.ค. เคส 09: goal ที่ได้คือ `"พักยืดเหยียด 5 นาที"` — ปลอดภัยแต่ **ยังไม่ผูกกับตารางของใครเลย**

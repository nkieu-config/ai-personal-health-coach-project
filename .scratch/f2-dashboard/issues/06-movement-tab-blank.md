## ต้นเหตุ — A ทำพังเอง ไม่ใช่ของ 🟦

commit `f7f5473` (ท้าย PR #46) ตั้งใจเปลี่ยนกราฟเคลื่อนไหวจาก stacked bar แยกชนิด → แท่งนาทีรวมสีเดียว เพราะของเดิมเอานาทีรวมหารเฉลี่ยทุกชนิด = กราฟโกหก · **แก้ไป 2 ใน 3 ชั้น ลืมชั้นที่ 3:**

| ชั้น | ทำแล้ว? |
| --- | --- |
| `processedData` — ลบ `walk/stretch/stairs/bike/sport` ใส่ `movementMinutes` แทน | ✅ |
| `chartConfig` — ลบ key รายชนิด ใส่ `movementMinutes` | ✅ |
| **JSX `<Bar>` 5 ตัว — ยังชี้ `dataKey="walk"` … `"sport"` อยู่เหมือนเดิม** | ❌ **ลืม** |

พังซ้อนสองชั้น: `dataKey="walk"` หาฟิลด์ใน data ไม่เจอ → `undefined` → แท่งสูง 0 · และ `fill="var(--color-walk)"` ก็ตายด้วย เพราะ `ChartContainer` ปั๊ม `--color-*` จาก key ใน `chartConfig` (`ui/chart.tsx:85`) ซึ่งถูกลบไปแล้ว → ไม่มีทั้งข้อมูลและสี

commit message และคอมเมนต์ปิด F2-02 **เขียนไว้ว่าแก้แล้ว** ทั้งคู่ — เขียนสเปกถูก เขียน data ถูก แต่ไม่ได้แตะ JSX จริง · prettier จัดบรรทัด `<Bar dataKey="sport">` ใหม่พอดี ทำให้ diff **ดูเหมือน** แตะแล้ว

## ทำไม CI ไม่จับ

- `tsc` — recharts รับ `dataKey` เป็น `string` เฉย ๆ ไม่ผูกกับ type ของ data → `dataKey="walk"` ถูกต้องตามชนิดทุกประการ
- **e2e — ไม่มีเทสต์แตะกราฟนี้เลยแม้แต่ตัวเดียว** (`grep pillar e2e/` = 0) ไม่มีแม้แต่เช็คหัวการ์ด
- screenshot ที่ A แนบเป็นหลักฐานตอนปิด F2-02 — ถ่ายแท็บ default (นอน) ซึ่งปกติ **ไม่เคยกดแท็บที่ตัวเองเพิ่งแก้**

เคสเดียวกับบั๊ก `flex-1` ใน INFRA-18 เป๊ะ ๆ: อ้างว่าตรวจแล้วโดยไม่ได้ออกแรงกดของที่ตัวเองเพิ่งเปลี่ยน
